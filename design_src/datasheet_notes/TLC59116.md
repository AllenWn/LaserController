# TLC59116

Source PDF: `design_src/datasheet/tlc59116.pdf`

Firmware-relevant summary for the **TLC59116** 16-channel I²C LED sink driver used on the `Auxiliary board`.

## 1. What it is

- 16-channel **constant-current LED sink driver**
- Controlled over **I²C**
- Intended to drive LED channels by sinking current on `OUT0..OUT15`

## 2. Operating range relevant to firmware

From the viewed datasheet pages:

- Supply voltage: **3.0 V to 5.5 V**
- Max output voltage: up to **17 V**
- Output current range depends on `REXT` and configuration

In this project:

- logic supply is `VDD_3V3`
- LED anode rail is boosted locally to `VDD_5V0`

## 3. Pins relevant to this design

From pin configuration / pin functions:

- `REXT`
  - external resistor sets output current
- `A0`, `A1`, `A2`, `A3`
  - address inputs
- `OUT0..OUT15`
  - constant-current outputs
- `RESET`
  - **active-low reset input**
- `SCL`
  - serial clock input
- `SDA`
  - serial data input/output
- `VCC`
  - power supply
- `GND`
  - ground

## 4. Addressing meaning

The datasheet supports:

- 4 hardware address pins (`A0..A3`)
- plus software-programmable addressing behavior

In this project’s auxiliary schematic:

- `A0..A3` are strapped low

Firmware implication:

- hardware address selection is fixed in this board revision
- the LED driver sits on the shared auxiliary I²C bus

## 5. Output behavior

Important conceptual point:

- `OUT0..OUT15` are **sink outputs**
- the device does not source LED current
- the LED anodes are tied to a positive rail, and the TLC59116 sinks current to light them

That matches the auxiliary schematic:

- LED anodes go to `VDD_5V0`
- `OUTx` lines go to LED cathode-side current sinks

## 6. Reset / logic behavior

Relevant logic fact from datasheet:

- `RESET` is **active low**

So if firmware or hardware ever controls/reset-monitors this line, low means reset asserted.

## 7. Pins used in this design

From the auxiliary schematic:

- `SCL` -> shared auxiliary I²C bus
- `SDA` -> shared auxiliary I²C bus
- `A0..A3` -> tied low
- `REXT` -> external resistor to ground
- `OUT0..OUT7` -> visibly used for:
  - RGB LED channels
  - multiple red LED channels
- higher outputs are currently not visibly used

## 8. What firmware should care about

For firmware, this device is simple compared with the safety-critical subsystems.

Main responsibilities:

- initialize / configure the LED driver if needed
- set channel PWM / brightness values
- control LED indications for status / UI

It is **not** part of the laser emission permit chain.

## 9. Design-specific conclusions

- TLC59116 is a UI/indicator device on the auxiliary I²C bus
- It should be treated as a **non-safety-critical output peripheral**
- If the bus is shared with MCP23017 and DRV2605, firmware needs clean bus ownership/arbitration, but a TLC59116 failure should not directly compromise the laser safety core
