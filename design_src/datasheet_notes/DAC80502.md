# DAC80502 (Dual DAC, DACx0502 family)

Source: `design_src/datasheet/dac80502.pdf`

## Interfaces
- Supports SPI or I2C, selected by `SPI2C` pin.
- In I2C mode: 2-wire `SCL` / `SDA` plus address selection on `A0` (pin shared name `SYNC/A0` in the datasheet).
- 7-bit addressing supported (10-bit addressing not supported).

## I2C Addressing
- Factory preset MSBs: `1001`
- `A0` selects the remaining address bits; datasheet notes A0 can be tied to `VDD`, `AGND`, `SCL`, or `SDA` for four addresses.
- This PCB note (schematic): A0 pulled LOW for slave address `1001000b` (7-bit) = `0x48`.

## Pinout (per datasheet pin table)
- `VDD` (pin 1)
- `VOUTA` (pin 2)
- `RSTSEL` (pin 3)
- `AGND` (pin 4)
- `SPI2C` (pin 5)
- `SCLK/SCL` (pin 6)
- `SYNC/A0` (pin 7)
- `SDIN/SDA` (pin 8)
- `VOUTB` (pin 9)
- `VREFIO` (pin 10)

## Output / Reference Configuration (high level)
- Output depends on DAC code and reference configuration.
- Datasheet describes:
  - `REF-DIV` (divider) selection
  - `BUFF-GAIN` selection per channel
  - Internal vs external reference (via `VREFIO` / reference power-down control)

## Firmware-Relevant Notes For This PCB
- `SCL` and `SDA` are routed to the ESP32.
- `VOUTA` goes to LD controller setpoint input; `VOUTB` goes to TEC controller setpoint input.
