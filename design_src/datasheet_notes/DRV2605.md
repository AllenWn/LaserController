# DRV2605

Source PDF: `design_src/datasheet/drv2605.pdf`

Firmware-oriented notes for the **DRV2605** haptic driver used in this project.

## 1. Device role

- I²C-controlled haptic driver
- Supports both:
  - `ERM` actuators
  - `LRA` actuators
- Includes waveform library, smart-loop control, auto-calibration, and real-time playback modes

In this project it is a user-feedback peripheral, not a laser-safety core block.

## 2. Relevant pins

From the datasheet pin table and board usage:

- `EN` — device enable
- `IN/TRIG` — trigger / input pin with mode-dependent meaning
- `SCL` — I²C clock
- `SDA` — I²C data
- `OUT+`, `OUT-` — actuator drive outputs
- `VDD`, `GND`

## 3. Interface and address

- Bus type: `I²C`
- Datasheet address: `0x5A` (7-bit)

Firmware implication:

- This is a proper bus peripheral
- It should be initialized over I²C before use

## 4. `EN` semantics

Datasheet meaning:

- `EN high` -> device enabled
- `EN low` -> device disabled / low-power

Project implication:

- `EN` is a hard gate for the haptic driver
- On the auxiliary board, `EN` is not necessarily driven directly by the MCU; it may be gated by the MCP23017 expander

## 5. `IN/TRIG` semantics

This pin is mode-dependent. The datasheet uses it differently in different operating modes:

- external trigger edge mode
- external trigger level mode
- PWM/analog input mode
- audio-to-haptics mode
- real-time playback related control flow

Firmware implication:

- Do not assume `IN/TRIG` always means “high = vibrate”
- Its meaning depends on the selected register mode

## 6. ERM vs LRA support

Datasheet features relevant to firmware:

- Auto-resonance support for `LRA`
- Calibration and diagnostics for both `ERM` and `LRA`
- Different register configuration paths for ERM and LRA operation

Project implication:

- Firmware must know which actuator type is actually stuffed on the board
- Board-level actuator choice changes the correct initialization sequence

## 7. Practical firmware modes

For this project, the most likely useful modes are:

- internal waveform playback for simple feedback patterns
- RTP mode for direct host-driven amplitude control

This is enough for:

- ready indication
- fault indication
- trigger acknowledgement

## 8. Project-specific interpretation

The DRV2605 should be treated as:

- non-safety-critical
- user-feedback only
- allowed to fail without changing the laser interlock decision
