# TPS2121

Source PDF: `design_src/datasheet/tps2121.pdf`

Firmware-relevant summary for the **TPS2121** power mux used on the `BMS board`.

## 1. What it is

- Priority power multiplexer / source selector
- Selects between two input sources:
  - `IN1`
  - `IN2`
- Drives one common output:
  - `OUT`

In this project, the BMS board uses it to select between:

- `VBUS`
- `VBAT`

and produce:

- `VSYS`

## 2. Important operating range

From the viewed datasheet pages:

- Input voltage range: **2.8 V to 22 V**
- Output voltage range: up to **22 V**
- Status/control pins are low-voltage logic domain pins

The datasheet also states:

- `TPS2121` continuous input current is higher than `TPS2120`
- on-resistance is lower than `TPS2120`

## 3. Pins relevant to TPS2121 package used here

From the datasheet `TPS2121 (RUX)` pin configuration:

- `IN1` (pin 7)
  - power input for source 1
- `IN2` (pin 2)
  - power input for source 2
- `OUT` (pins 1, 8)
  - power output
- `ST` (pin 9)
  - status output indicating which channel is selected
- `ILM` (pin 10)
  - output current limit setting pin
- `SS` (pin 11)
  - input setting delay / output soft-start timing
- `GND` (pin 12)
  - ground
- `PR1` (pin 6)
  - priority select / comparator enable related pin
- `OV1` (pin 5)
  - overvoltage supervisor input for `IN1`
- `OV2` (pin 4)
  - overvoltage supervisor input for `IN2`
- `CP2` (pin 3)
  - comparator / switchover related support pin on TPS2121

## 4. Logic meaning of important pins

- `ST`
  - **status output**
  - indicates which source is currently selected
- `PR1`
  - used to configure priority behavior
- `OV1`, `OV2`
  - configure or supervise input overvoltage behavior
- `ILM`
  - resistor-programmed current limit
- `SS`
  - sets switchover delay / soft-start timing

These are mostly **analog configuration / supervisory** pins, not MCU digital control pins in this board.

## 5. Key datasheet electrical behavior

From viewed specs:

- `TPS2121` fast switchover time is on the order of **microseconds**
- UVLO thresholds are in the ~2.5 V region
- current limit is resistor-programmed

This is important because it means:

- source selection happens in hardware
- this is not something firmware is expected to react to in real time

## 6. What the project schematic configures

The BMS sheet note explicitly states:

1. `TPS2121` is configured to prioritize **VBUS over VBAT** whenever `VBUS > 5V`
2. `ILIM` is set by `18k7`
3. `OVx` is set to approximately `21.5V` for both inputs

So the power-source priority is hard-configured in hardware.

## 7. Firmware meaning

For firmware, the most important conclusion is:

- `TPS2121` is **not** an I²C/SPI peripheral
- the ESP32 does not directly configure it in normal operation
- it is a hardware power-path component

Therefore, firmware should think about it as:

- part of the system power architecture
- not a device driver target

## 8. Relevance to this project

The BMS board uses TPS2121 to create `VSYS` for the main board.

That means:

- the main firmware receives the effect of source selection indirectly
- but there is no evidence in the current schematic set that the MCU directly controls this mux

## 9. Design-specific conclusions

- This is a **system power-path device**, not a firmware-controlled feature block
- It matters to firmware because it explains:
  - where `VSYS` comes from
  - why the main board may run from either `VBUS` or `VBAT`
- It does **not** currently add a direct MCU communication interface
