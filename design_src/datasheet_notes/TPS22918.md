# TPS22918

Source PDF: `design_src/datasheet/tps22918.pdf`

Firmware-oriented notes for the **TPS22918** load switch used on the `MainBoard`.

## 1. Device role

- Load switch
- Supports controlled turn-on slew rate
- Supports configurable quick output discharge (`QOD`)

In this project it is used on the visible-light switch path, not as the primary NIR laser safety gate.

## 2. Relevant pins

From the datasheet pin function table:

- `VIN`
- `VOUT`
- `ON`
- `CT`
- `QOD`
- `GND`

## 3. `ON` semantics

Datasheet meaning:

- `ON` is an active-high switch control input
- must not be left floating

Firmware implication:

- `HIGH` -> switch enabled
- `LOW` -> switch disabled

## 4. `CT` semantics

Datasheet meaning:

- `CT` sets the output slew rate / rise time
- can be left floating depending on application

Project implication:

- `CT` is a hardware-tuning node
- It is not a runtime firmware interface

## 5. `QOD` semantics

Datasheet meaning:

- `QOD` controls quick-output-discharge behavior
- Can be enabled in multiple ways:
  - external resistor between `VOUT` and `QOD`
  - tying `QOD` to `VOUT`
  - disabling by leaving the pin floating

Project implication:

- `QOD` changes how fast the switched output collapses when disabled
- This matters for hardware behavior, but is not usually MCU-driven directly

## 6. Project-specific usage

From the mainboard design:

- `ON` is driven by `GN_LD_EN`
- The switched source selection is set by resistor stuffing, not by firmware protocol

Firmware implication:

- This device behaves like a simple active-high switched output
- It should be modeled as a functional output path, not a communications peripheral

## 7. Project-specific interpretation

For firmware architecture:

- The TPS22918 belongs to auxiliary output control
- It does not replace the primary laser interlock path based on `SBDN`
- Its default state and polarity still matter during bring-up and fault handling
