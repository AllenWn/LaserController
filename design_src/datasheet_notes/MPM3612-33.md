# MPM3612-33

Source PDF: `design_src/datasheet/MPM3612GLQ-33.pdf`

Firmware-oriented notes for the **MPM3612-33** buck regulator used for the logic rail.

## 1. Device role

- Fixed-output buck regulator module
- Generates the board `3.3 V` rail
- Usually part of the always-on logic power tree

## 2. Relevant pins

From the datasheet:

- `EN` — enable input
- `PG` — power-good output

## 3. `EN` semantics

Datasheet meaning:

- `EN high` -> regulator on
- `EN low` -> regulator off
- The pin is a digital control input and should not be floated

Project implication:

- If firmware or external hardware controls this rail, it is a true power enable

## 4. `PG` semantics

Datasheet meaning:

- `PG` is an `open-drain` power-good output
- It requires a pull-up resistor

Datasheet behavior also calls out:

- `PG` goes high after the output is ready
- `PG` pulls low immediately on disable or certain fault conditions
- documented rise/fall delays are on the order of `120 µs` / `50 µs`

Practical firmware interpretation:

- `HIGH` -> 3.3 V rail healthy
- `LOW` -> rail not healthy, regulator disabled, startup, or protection event

## 5. Why this matters to firmware

Even if the MCU does not actively sequence this rail during normal runtime, the regulator explains:

- where the logic domain comes from
- what a logic-rail `PG` signal means
- how upstream power issues may appear to the system

## 6. Project-specific interpretation

The MPM3612-33 is not part of the laser-permit decision directly.

It is part of the platform power foundation:

- if this rail is wrong, the MCU and peripherals are already outside normal assumptions
- therefore this device is more relevant to bring-up, reset behavior, and platform stability than to the runtime laser interlock itself
